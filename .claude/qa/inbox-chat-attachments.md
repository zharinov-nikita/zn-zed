## 3. Карточка: содержимое и длинный текст

- [ ] Задача с длинным заголовком (несколько предложений) — заголовок обрезается, карточка не растёт

- [ ] Несколько карточек подряд стоят плотно: без пустых строк между ними, и в черновике,

- [ ] На карточке видны поля задачи: чип списка, чипы тегов, возраст, счётчик подзадач (`2/5`),
      файл-контекст, вложения — те же, что в строке панели.
- [ ] Скрыть поле в меню «Fields» панели (например, теги) → на карточке в чате оно тоже пропало.
- [ ] Задача без полей (только заголовок) — карточка выглядит нормально, пустой строки полей нет.

## 4. Мульти-выделение

- [ ] `Ctrl+клик` по задаче → строка подсвечена, detail view не открылся.
- [ ] `Ctrl+клик` по ней же → подсветка снята.

- [ ] `Shift+клик` по строке ниже → выделен диапазон от последней кликнутой.
- [ ] Обычный клик по заголовку → открывается detail view.
- [ ] `Escape` в панели → выделение снято.
- [ ] Перетаскивание одной из выделенных строк в чат → три карточки подряд, в порядке списка,
      каждая на своей строке.
- [ ] Перетаскивание невыделенной строки → только она.

## 5. Перетаскивание внутри панели (регрессия)

- [ ] Сортировка Manual: дроп строки на строку меняет порядок.
- [ ] Дроп пачки из 2-3 выделенных на строку → переносятся все, относительный порядок сохранён.
- [ ] Режим «By list»: дроп на группу меняет список у всех перетащенных задач.

## 6. `@task` в чате

- [ ] `@` в поле ввода → в списке категорий есть «Inbox Tasks».
- [ ] Выбор категории → задачи в порядке панели, обработанных (cleared) в списке нет.
- [ ] Ввод части заголовка фильтрует список.
- [ ] Enter по задаче → та же карточка, что даёт перетаскивание; набранный `@…`-текст исчез.
- [ ] `@task` **после текста в той же строке** → текст остаётся видимым, карточка встаёт на
      следующей строке (блок не съедает строку).
- [ ] Уже вложенная задача в списке `@task` не показывается.
- [ ] При пустом инбоксе категория не показывается.

## 7. Клик и реактивность

- [ ] Наведение на карточку → курсор pointer, лёгкая подсветка.
- [ ] Клик по карточке в черновике → фокус на inbox-панели, открыт detail view этой задачи
      (было: клик не работал).
- [ ] Клик по карточке в отправленном сообщении → то же самое.
- [ ] Переименование задачи в панели → карточка в черновике сразу показывает новый заголовок.
- [ ] Правка тегов/списка/чек-листа → строка полей на карточке обновилась.
- [ ] Отметка задачи выполненной → карточка приглушена и помечена «done».
- [ ] То же для карточки в отправленном сообщении.

## 8. Удаление задачи

- [ ] Прикрепить задачу в черновик, **не отправляя**, удалить её в панели → карточка и её строка

- [ ] Две карточки в черновике, удалить одну задачу → ушла только она.
- [ ] Отправить сообщение с задачей, потом удалить задачу → в истории карточка осталась,
      заголовок перечёркнут, пометка «deleted».

## 9. Drag из detail view

- [ ] Открыть задачу (detail view) → в шапке слева от скрепки есть ручка с иконкой треда,
      тултип «Drag into the AI chat».
- [ ] Тянуть её в чат → ghost с заголовком, в поле ввода появляется карточка этой задачи.
- [ ] Выделение текста в теле задачи мышью работает как раньше (drag-ручка ему не мешает).

## 10. Редактирование отправленного сообщения (был баг)

- [ ] Отправить сообщение с задачей → «Edit message» → **карточка на месте**, не пропала.
- [ ] Отредактировать текст вокруг карточки и отправить снова → задача по-прежнему приложена.
- [ ] Удалить карточку в режиме редактирования → уходит вместе со ссылкой.

## 11. Что уходит модели

- [ ] Прикрепить задачу → поправить её текст → отправить: агент видит новую версию, а не ту,
      что была при перетаскивании.
- [ ] Спросить у агента id вложенной задачи → отвечает корректным id.
- [ ] Попросить агента дополнить задачу → вызывает `inbox_update_item`, правка видна в панели
      и на карточке.
- [ ] Отправка карточки удалённой задачи → внятная ошибка, а не пустой блок.

## 12. Ревью-фиксы (новое)

- [ ] Скопировать текст сообщения с карточкой (Ctrl+C по выделению с ссылкой `[@…](zed:///agent/inbox-item…)`),
      вставить **посреди строки** другого черновика → текст строки не съеден, карточка встала
      на свою строку.
- [ ] Печатать в заголовке задачи в detail view, пока агент правит эту задачу по MCP →
      набираемый текст не затирается (внешняя правка подтянется после ухода фокуса).
- [ ] Открыть попап удаления вложения в detail view → правка ДРУГОЙ задачи по MCP не закрывает попап.
- [ ] Приложить задачу после текста (`текст @task…`), удалить задачу в панели → в черновике
      не остаётся пустой строки.
- [ ] Выделить A, B, C и бросить выделение на B (внутри выделения) → порядок не меняется (no-op).
- [ ] Выделить задачи → включить тег-фильтр → выделение сброшено (drag не тащит невидимое).
- [ ] `Ctrl+клик` снять выделение со строки → `Shift+клик` ниже → снятая строка не возвращается.
